一个小型且强大的 Angular 状态管理类库。

为什么？

之前我们写过一篇 Angular 真的需要状态管理么？ 的文章，在 Angular 框架下，通过Service和Observable组合可以轻松管理前端应用的状态，但是RxJS太过于抽象和难学，所以就导致部分开发人很难驾驭RxJS，容易在组件中直接修改状态，为了约束开发者的行为，同时提供统一的 API 简化业务操作，所以我们封装了一个小型的状态管理类库，让普通的开发人员也能很好的管理状态。

特性

  • 完美结合 Angular 的设计理念，Store即服务
  • 领域驱动，多Store模型，每个Store属于一个领域，状态存储和操作聚合一起
  • 简单易上手的 API，无需过多学习成本
